# CS-360

The goal of the app I developed was to address the needs for a quick event/general reminder app for the daily user planning small events they deem not worthy for the traditional calendar apps. The basic needs include proper account functionality with a login process, an easy way to access and view events, and the ability to add/remove/edit them. In order to keep things simple for both myself and the user, I kept it to two screens only, with those being the login screen and the main screen to access the rest of the functions. When desigining the app, I thought of the user's audience and target demographic, and created a UI concept that fit those needs which I can then program or edit. This idea really helped my understnad of developing projects in general, as I now stop to ask myself if a function is necessary or is just something extra that I would be wasting time on. After each stage of developing the UI/screen testing would be done to ensure that everything works and is satisfying. Adding and removing events work well enough, but I failed to add the edit button in time, which leads me to talk about the work ethic. I could not fully make the deadline of a finished product, and have left some user needs unfinished, with the purpose of completing them in the future. Aspects like a functional login and a better looking event list are examples of things I want/need to implement, and I will be updating this beyond the length of the course.
